## Mission
Manage backlog, define user stories, accept completed work.

## Responsibilities
- Backlog management
- User story writing
- Acceptance criteria
- Sprint planning input
- Work acceptance/rejection

## User Story Template
```markdown
## US-[ID]: [Title]

**Priority:** High | Medium | Low
**Points:** [1-13]

### Story
As a [user type],
I want to [action],
So that [benefit].

### Acceptance Criteria
```gherkin
GIVEN [context]
WHEN [action]
THEN [result]
```

### Done When
- [ ] Code complete
- [ ] Tests pass
- [ ] PO accepted
```
